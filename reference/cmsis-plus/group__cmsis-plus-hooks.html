Detailed Description

CMSIS++ application hooks.

Applications using CMSIS++ can customise the behaviour of the system or intercept several events by redefining several hooks.

All hooks have weak default definitions, used if the application does not redefine the function.

bool os_idle_enter_power_saving_mode_hook ( void  )

Hook to enter a power saving mode.

Parameters
None
Return values
trueThe hook entered a power saving mode.
falseThe hook did not enter a power saving mode.

The hook must check an application specific condition to determine if it is required to enter a power saving mode, and, if necessary, actually enter the desired power saving mode.

The application must ensure that all interrupts associated with the external event used to awake the device are enabled. Usually the RTC is used for this purpose, but also other devices, like USB, GPIO pins, etc may be used to end the power saving mode.

This function is executed by the idle thread on each iteration, and must be reasonably complex.

If the user function decides not to enter a power saving mode, it must return false, which will make the idle thread proceed as usual, by entering a shallow sleep waiting for the next interrupt.
