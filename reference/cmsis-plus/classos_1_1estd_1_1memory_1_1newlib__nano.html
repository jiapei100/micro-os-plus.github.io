Detailed Description

Memory resource implementing newlib nano policies.

This memory manager was inspired from the newlib nano implementation of malloc() & free(). It is not particularly great, on the contrary.

The free list is kept ordered by addresses, which means deallocate() will need to traverse most of the free list to enqueue blocks allocated lately.

allocate() too must traverse a good part of it, trying to identify a block with the required size.
