20 /*
21  * This file is part of the CMSIS++ proposal, intended as a CMSIS
22  * replacement for C++ applications.
23  *
24  * The code is inspired by ARM CMSIS cmsis_os.h file, v1.02,
25  * and tries to remain functionally close to the CMSIS specifications.
26  *
27  * Major improvements:
28  * - no more macros required to define objects
29  * - for applications that require it, allows fully static
30  * memory allocations
31  * - very close to POSIX (IEEE Std 1003.1, 2013 Edition)
32  * http://pubs.opengroup.org/onlinepubs/9699919799/nframe.html
33  * - specifically designed as a convenient implementation for
34  * C++ standard thread library (ISO/IEC 14882:2011)
35  * - improved usability, by defining both simple (using defaults)
36  * and complex (using attributes) object constructors (feature
37  * inspired by POSIX threads attributes)
38  * - improved readability with explicit three-fold waiting functions
39  * (for example: lock(), try-lock(), timed-lock(), similar to POSIX threads)
40  * - POSIX condition variable added
41  * - versatile clocks added (Systick_clock, Realtime_clock)
42  * - standard POSIX errors definitions used
43  */
